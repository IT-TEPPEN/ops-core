## はじめに

### OpsCoreとは

OpsCoreは、GitHubやGitLabなどの外部リポジトリに保存された運用手順書（Markdownファイル）を集約し、Web上で閲覧・実行できるシステムです。

### 主な機能

1. **ドキュメント管理**: 手順書の公開・バージョン管理
2. **変数入力**: 手順書実行時のパラメータ入力
3. **作業証跡**: 作業履歴の記録と共有
4. **グループ管理**: チームでのアクセス制御

### ユーザーの役割

#### 管理者（Admin）
- 全てのリポジトリとドキュメントの管理
- ユーザーとグループの管理
- システム設定の変更

#### 一般ユーザー（User）
- 公開ドキュメントの閲覧
- 自分が作成したドキュメントの管理
- 作業証跡の作成と閲覧

## 基本的な使い方

### 1. ログイン

（将来実装予定）

```
1. OpsCoreのURLにアクセス
2. ユーザー名とパスワードを入力
3. 「ログイン」ボタンをクリック
```

### 2. ホーム画面

ログイン後、ホーム画面が表示されます：

- **リポジトリ一覧**: 登録済みのGitリポジトリ
- **ドキュメント一覧**: 公開中の手順書
- **最近の作業証跡**: 自分の最近の作業履歴

### 3. ドキュメントの閲覧

```
1. ドキュメント一覧から閲覧したい手順書を選択
2. 必要に応じて変数を入力（左ペイン）
3. 手順書の内容を確認（中央ペイン）
4. 必要に応じて作業証跡を記録（右ペイン）
```

### 4. 検索機能

```
1. 上部の検索バーにキーワードを入力
2. タグやカテゴリでフィルタリング
3. 検索結果から目的の手順書を選択
```

## よくある質問

### Q1: ドキュメントが表示されない

**A**: 以下を確認してください：
- ドキュメントが公開されているか
- 自分に閲覧権限があるか
- ブラウザのキャッシュをクリアしてみる

### Q2: 変数の入力方法がわからない

**A**: [変数入力機能ガイド](./variable-input.md)を参照してください。各変数の説明とデフォルト値が表示されています。

### Q3: 作業証跡を他のユーザーと共有したい

**A**: 作業証跡詳細画面の「共有」ボタンから、共有したいユーザーまたはグループを選択できます。

### Q4: パスワードを忘れた

**A**: ログイン画面の「パスワードを忘れた場合」リンクから、パスワードリセット手続きを行ってください。（将来実装予定）

### Q5: エラーが発生した

**A**: 以下の情報をメモして、管理者に連絡してください：
- エラーメッセージの内容
- エラーが発生した操作
- 発生日時

## 使用上の注意

### セキュリティ

- パスワードは定期的に変更してください
- 共有PCでは必ずログアウトしてください
- 作業証跡に機密情報を含めないでください

### パフォーマンス

- 大容量の画像ファイルの添付は避けてください（推奨: 2MB以下）
- 同時に複数のドキュメントを開くとブラウザが重くなる場合があります

### データ保存

- 作業証跡は自動保存されません。定期的に「保存」ボタンをクリックしてください
- ブラウザを閉じる前に、未保存の変更がないか確認してください
